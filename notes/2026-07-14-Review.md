## 5. case_when()

✅ Correct

```r
case_when(
    SalePrice < 150000 ~ "Low",
    SalePrice < 250000 ~ "Middle",
    TRUE ~ "High"
)
```

Correct understanding that conditions are evaluated from top to bottom.

No need to explicitly write

```r
SalePrice >= 150000 &
```

because the previous condition has already excluded those rows.

## 7. Missing Values

⚠⚠ syntax mistake ⚠⚠

Incorrect:

```r
mean(train$LotFrontage, NA=True)
```

Correct:

```r
mean(train$LotFrontage, **na.rm = TRUE**) # 결측값을 제외하고 평균을 계산하는 코드
```

Important:

* `NA` is a missing value.
* `na.rm` means "remove missing values."

This argument appears frequently in R.

# Minor Mistakes

### 1.

```r
SalePrice_10k %in% names(train)
```

This causes an error because `SalePrice_10k` is interpreted as an object.

Correct:

```r
**"SalePrice_10k" %in% names(train) **
```

or

```r
names(train_price)
```

---

### 2.

```r
class(PriceGroup)
```

`PriceGroup` is not an independent object.

⚠⚠⚠Correct:

```r
class(train_grouped$PriceGroup)
```

---

### 3.

```r
mean_price, median_price
```

⚠⚠ R does not print two objects separated by commas.

Use

```r
mean_price
median_price
```

or

```r
c(mean_price, median_price)
```

---

### 4.

⚠⚠ Remember

```r
na.rm = TRUE
```

not

```r
NA = True
```

This is one of the most common beginner mistakes.
